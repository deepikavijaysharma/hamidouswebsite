Design a cloud Solution

Category: Oppurtunity Execution

Description

This Solution-Engineering-as-a-Service includes:
  • Collaboratively working with a customer to turn their Cloud Vision into a robust Solution Presentation and Oracle Sales Proposal.
  • Producing a sales proposal, including developing logical and physical architectures, sizing current and future workloads, recommending phased implementation roadmaps, creating the intake provisioning document, solution presentation, and contributing to the Oracle Sales Proposal.
  • This service typically uses ECAL Stages 2-4 as the underlying execution framework.

SE Role Engagement

Roles:
  • ECAs lead this engagement and are responsible for the majority of the deliverables.
  • Platform Specialists and Hub SEs work on Portfolio and Workload Analysis efforts, contribute to the physical architecture designs, and provide foundation services as needed (e.g. Customer Discovery of current state IT Portfolio, Customer Demos, Sizing/Quoting/Config).
  • This service may spawn the Build a POC service.
  • Business Value Consultants support any value-based activities and deliverables, which can also fall under the Build a Business Case service.
  • CSMs are responsible for ECAL Migration and Adoption Planning activities and deliverables, and the subsequent Adopt a Cloud Solution service.
  • For more information, see our ECAL website.

How do you request this service?

Initially this request will be initiated by ECA via https://apex.oraclecorp.com/pls/apex/f?p=21625:1:100420461220824
Service based model in SCRM is target to be enabled on 12/1/2017.

Customer Discovery

Category: Foundational SE Services

Description

This Solution-Engineering-as-a-Service includes:
  • Collaboratively working with a customer to analyze specific business/technical use cases and associated opportunities and challenges.
  • This process is highly consultative, focusing on the customers business/technical goals as well as existing processes, technologies, systems, personnel and business environment.
  • This builds valuable customer trust, but also uniquely positions Oracle to business-justify and cost-justify immediate implementation of a solution – specifically immediate implementation of an Oracle Cloud solution.
  • Deliverables include a presentation outlining use cases, goals, challenges, prescription analyses and benefits. Deliverables also include a solution roadmap and Oracle sales proposal.

SE Role Engagement

Roles:
  • ECAs lead initial steps in this engagement and are responsible for the majority of the early deliverables.
  • Platform Specialists from the GEO Field support all phases closely, coordinating customer events and providing seamless handoff to solution and proposal.

How do you request this service?

Initially this request will be initiated by ECA via https://apex.oraclecorp.com/pls/apex/f?p=21625:1:100420461220824
Service based model in SCRM is target to be enabled on 12/1/2017.

Platform Assessments

Category: Foundational SE Services

Description

This Solution-Engineering-as-a-Service includes:
  • Working with a customer to analyze their current-state platform/system topology and environment.
  • Identify challenges and problems and associated impacts/costs.
  • Identify low-hanging-fruit opportunities for improvement as well as a full, future-state vision to address all platform needs.
  • Define benefits, and provide cost justifications for short and long-term solutions.
  • Provide a complete roadmap for platform rationalization.
  • Perform all work required to produce the sales proposal, including physical architectures, sizing for current and future workloads, and phased implementation recommendations.

SE Role Engagement

Roles:
  • Platform specialists from the GEO Field will lead these engagements working in conjunction with ECAs.
  • Platform specialists will focus on specific sizing and implementation issues, supported where appropriate by Hub SEs.
  • ECAs will focus on overall architectural design.

How do you request this service?

Service based model in SCRM is target to be enabled on 12/1/2017.

Hands-on Workshops

Category: Foundational SE Services

Description

This Solution-Engineering-as-a-Service includes:
  • Providing a technical hands-on experience to a customers technical personnel on a single or set of cloud services.
